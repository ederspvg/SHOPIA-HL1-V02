Manual: Resolução de Problemas na Importação de CTE do Informa para o Protheus 1. Introdução Este manual fornece orientação detalhada para a equipe de suporte de TI na resolução de problemas relacionados à rotina customizada do Protheus WFLUFT10. Esta rotina é responsável pela importação de dados de CTE (Conhecimento de Transporte Eletrônico) do Informa para o Protheus. A rotina processa arquivos de texto, valida os dados e importa as informações do CTE para as tabelas relevantes do Protheus. 2. Visão Geral da Rotina A rotina WFLUFT10 automatiza a importação de dados de CTE de arquivos de texto do Informa para o sistema Protheus ERP. Aqui está uma descrição detalhada de suas funções: ● Entrada de Arquivos: A rotina lê os dados do CTE de arquivos de texto localizados no diretório \Int_Informa\. ● Validação de Dados: ○ Valida o formato e o conteúdo de cada linha, garantindo a integridade dos dados. ○ ○ As validações específicas incluem a verificação de um registro de cabeçalho (registro '1'), a verificação dos tipos de registro ('1' para cabeçalho, '2' para detalhes, '3' para cancelamento), a validação dos formatos de data e a garantia de registros de detalhes completos. ○ ● Verificação de Duplicidade: A rotina verifica se há entradas de CTE duplicadas no Protheus para evitar redundância de dados. ● Tratamento de Clientes: ○ A rotina cria novos registros de clientes no Protheus, caso eles não existam. ○ ● Importação de Dados: ○ As informações de cabeçalho e detalhe do CTE são importadas para as tabelas do Protheus (DT6, DTC). ○ ○ Os dados de composição do frete são importados para a tabela DT8. ○ ● Geração de Faturas: A rotina gera faturas de frete. ● Registro de Log: ○ A rotina registra meticulosamente a atividade de importação e os erros. ○ ○ Os logs são armazenados no diretório \Int_Informa\Logs\.
